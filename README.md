# Software Engineering Foundation Portfolio

This repository highlights the software development foundation behind my current Quality Engineering and Automation leadership profile.

Before moving into large-scale automation architecture and QA leadership, I worked in software development roles across banking and insurance platforms, contributing to system design, coding, batch processing, production issue analysis, and release support.

## What this portfolio showcases
- Mainframe and enterprise software development experience
- COBOL, SQL, CICS, JCL and batch-processing exposure
- Banking and insurance system delivery experience
- Production troubleshooting and issue-resolution mindset
- Design participation, impact analysis, and code validation
- Engineering depth that later strengthened my automation architecture and QE leadership approach

## Development Experience Areas

### Mainframe and Banking Systems
Worked on enterprise applications supporting business-critical customer and financial workflows.

### Coding and Program Enhancements
Contributed to new program development and enhancements using COBOL, SQL, and CICS.

### Batch and Release Support
Created and maintained JCL JOBs and PROCs, supported SIT and batch cycles, and participated in release validation.

### Production Analysis and Stability
Analyzed logs, investigated ABENDs, resolved production issues, and supported quick turnaround during critical phases.

### Design and Impact Analysis
Created functional designs, participated in design review discussions, and performed impact analysis for business and system changes.

## Why this matters
My software development foundation gave me a strong understanding of how enterprise systems are built, how failures occur, and how design choices affect quality. That foundation later helped me build stronger automation frameworks, backend validation strategies, and engineering-led quality practices.
